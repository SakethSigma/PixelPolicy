## Unified SFT schema (distillation/schema.py)
Every game's SFT row shares these columns (`sft_row`; legacy Wordle upgraded by `normalize_legacy`):
`game_name`, `game_no`, `round`, `valid`, `target`, `system`, `messages`, `completion`, `completion_no_think`, `has_think`, `episode`.
- `game_no` (registry `GAME_NUMBERS`): wordle=0, charcount=1, validity=2, anagram=3, endstart=4, rhyme=5, crossword=6, charset=7, mistakeid=8, tower=9, codebreaker=10, bullscows=11, consistency=12. ALL 13 BUILT (endstart was the last of original six; 10/11/12 added 2026-06-13). `GameSpec.require_think=True` on the reasoning games anagram, crossword & mistakeid (keep only correct traces that carry a `<think>` block). NOTE: the old `reject_think` flag is GONE — anagram was redone as a HIGH-EFFORT reasoning game (not direct/low-effort). push.py `_STEM_GAME` maps `crossword_sft`→(crossword,6), `charset_sft`→(charset,7), `mistakeid_sft`→(mistakeid,8).
- `valid`: the rejection gate. Single-turn games gate on CORRECTNESS ("correct" per `GameSpec.good_status`). WORDLE IS THE EXCEPTION: its `valid` is re-derived in push.py `load_rows` from `has_think` (format compliance — does the move carry a `<think>` block), NOT win/loss. So a reasoned move from a lost game is kept; a no-think move is dropped even if won. Wordle keeps `require_think=False` on its spec; the format check lives in push.py. (Changed 2026-06-13 from won-based; Wordle valid rows went 1,542→2,602, 476 no-think dropped.)
- Original Wordle dataset's `game` column was the episode index; push.py/reexport.py rename it to `episode` and add game_name/game_no.

(Prior count 60,162/59,520 — and older 55,162 / 42,832 — now stale; still present ONLY in new_game.md, the off-limits live tracker.)
- charset (game 7, programmatic, NO Claude): 12,000 rows via `programmatic.py --game charset`, all valid. Each challenge = 1 five-letter Wordle word + 1-3 non-five-letter words (2-4 total), even spread of 2/3/4-word; salted `charset` split. Ground truth pure Python: used=union of letters, unused=a-z minus used. Answer = `used (K): ...` / `unused (M): ...` UPPERCASE space-sep. programmatic.py `--game` choices now: charcount|validity|rhyme|charset.
- mistakeid (game 8, Claude-distilled reasoning, require_think): 330 episodes (165 mistake + 165 clean) at claude-sonnet-4-6 `--effort max` (xhigh NOT valid for this model; levels low/medium/high/max) → 317 valid (157 mistake + 160 clean; 13 wrong dropped), cost ≈$2.78 batch. Self-contained: reads committed `games/mistakeid/challenges.jsonl` (165 mistake + 1,498 clean boards, built by `games/mistakeid/build_challenges.py` from original Wordle teacher trajectories); `sample_targets` returns 50/50 mix (balanced max 165+165=330). NO target word — true errors derived from board feedback. grey mistake=reuses proven-absent letter; yellow mistake=re-places letter in slot already shown yellow. Answer = `mistakes: yes|no` + `position N, letter X, grey|yellow` lines.
- tower (game 9, programmatic, NO Claude, NO require_think): single-turn deduction game. 3 floors (1=bottom,3=top) x 2 rooms (Left/Right); 3 people, one per floor (bijection). Shown a proposed placement + per-person floor✓/x & room✓/x flags (Wordle-style); list EVERY consistent placement. Answer set provably 1 or 2 (exactly 2 iff all 3 floors wrong = the 2 derangements of 3; rooms always uniquely deducible). Whole distinct logic space only 1,920 → surface variety from a 60-name pool. Answer = numbered `solution N:` blocks, lines `Name: floor N, Left/Right`; scoring = set must EXACTLY equal consistent set. 5,000 rows via programmatic.py --game tower, all valid: ~3,343 single-solution + ~1,657 two-solution (~1/3 derangement rate). Package game-tower (game/render/client/server/play/tests, 20 tests), agent agents/tower/agent.py defines TowerAgent + TowerEnv. registry _tower_spec: good_status="correct", no require_think. NO vocab/corpus dep.
- programmatic.py `--game` choices now: charcount|validity|rhyme|charset|tower|endstart|codebreaker|bullscows|consistency. programmatic.py gained a MULTI-TURN generator: `_play_multiturn` reuses agents/rollout.py::run_episode with an injected solver as the `generate` callback (one SFT row per turn, $0, no Claude); `--max-rows` caps multi-turn output AT AN EPISODE BOUNDARY (whole episodes kept, round distribution stays unbiased).
- endstart (game 4, programmatic, single-turn MCQ): 6,000 rows; word1 + 5 candidates, pick the one whose first letter == word1's last letter (4 distractors start with other letters, options shuffled). GT word1[-1]==cand[0]. Answer `<answer>oasis</answer>`. Seeds from shared vocab (salted endstart split). Package game-endstart, agent EndstartAgent/EndstartEnv, _endstart_spec good_status=correct. 15 tests. Was the LAST of the original six.
- codebreaker (game 10, programmatic, MULTI-TURN Mastermind): secret 4 slots over A-F (repeats allowed), per-position ✓/-/x via Wordle's two-pass compute_feedback, max_rounds=12, good_status=won. Teacher = CodebreakerSolver (UNBIASED: random opening + uniformly random consistent code). Answer per turn `<guess>ACEF</guess>` no <think>. Capped 10,000 rows (~2,726 episodes, ~3.7 turns/ep) via `--episodes 5000 --max-rows 10000`. NO vocab dep. 16 tests.
- bullscows (game 11, programmatic, MULTI-TURN count feedback): secret 4 DISTINCT digits 0-9, feedback bulls+cows (COUNTS not tiles), max_rounds=10, good_status=won. Unbiased BullsCowsSolver. Answer `<guess>1234</guess>`. Capped 10,000 rows (~1,823 episodes, ~5.5 turns/ep) via `--max-rows 10000`. NO vocab dep. 16 tests.
- consistency (game 12, programmatic, single-turn yes/no): Wordle board (1-3 guesses + ✓/-/x) + candidate; answer `<answer>yes|no</answer>` = is candidate still possible. GT REUSES Wordle compute_feedback (dep on game-wordle): consistent iff compute_feedback(guess,candidate)==fb for every row. Balanced 50/50 (5k yes + 5k no), kept <4k tokens (max ~250). Positive-selection complement to mistakeid. 10,000 rows. 17 tests.
- anagram was REDONE: now 1,000 episodes at high effort (seed 0), all 1,000 correct, 932 valid (68 dropped for no `<think>`), cost ≈$1.78. The earlier 2,000-row low-effort/direct run was discarded.
- crossword (game 6) NOW BUILT: 1,500 episodes, claude-sonnet-4-6, --effort high → 1,415 correct / 1,415 valid (1,498/1,500 had `<think>`), split 750 Wordle-seed + 750 general-seed, cost ≈$4.96. Claude-distilled reasoning, `require_think`. Seeds = words w/ WordNet def, half Wordle-vocab + half general (NOT Wordle-only). Clue = def + length + ~half-masked pattern (deterministic word-seeded RNG). Seed word never in in-progress GameState (no `word` field; revealed in terminal `solution`).
- Programmatic defaults (`python -m distillation.programmatic --game X`, no Claude): charcount 14,000 (≥4,000 Wordle-vocab + 10,000 WordNet, len 3–20); validity 13,254 (6,627 valid + 6,627 invalid, 50/50; word universe = Wordle vocab 12,972, only 6,627 have WordNet defs); rhyme 10,000 (5,000 MCQ + 5,000 free).
- anagram: Claude-distilled via `batch_play.py --game anagram` (Batch API). vocab.txt = 85,909 words; meanings.jsonl = 79,564 entries.

## Producers
- Claude-distilled (reasoning, e.g. Wordle): `distillation/batch_play.py` (Anthropic Batch API). Re-shape raw dumps with `reexport.py`.
- Programmatic (no reasoning, e.g. charcount): `distillation/programmatic.py`.
- Combine + push: `push.py` (`--overwrite` wipes Hub shards after a schema change; `--dry-run` for stats). DEFAULT_INPUTS includes charcount.
- Adding a game to distillation = one GameSpec in `registry.py` (GAMES dict) + its number in GAME_NUMBERS.
